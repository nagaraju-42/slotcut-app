STEP 11: Set Up Backend (Railway) - OPTIONAL
──────────────────────────────────────────
For full functionality, you need the backend:

1. Go to https://railway.app
2. Create account (free with GitHub)
3. New Project → Import from GitHub
4. Select: https://github.com/nagaraju-42/slotcut-app
5. Configure:
   - Root directory: slotcut-backend
   - Build command: ./gradlew build
   - Start command: java -jar build/libs/slotcut-0.0.1.jar
6. Add environment variables:
   - SPRING_DATASOURCE_URL = your Supabase connection string
   - SPRING_DATASOURCE_USERNAME = postgres
   - SPRING_DATASOURCE_PASSWORD = your password

================================================================================
                      TROUBLESHOOTING
================================================================================

PROBLEM: "No Production Deployment" still shows
SOLUTION:
  1. Refresh page (Ctrl+R or Cmd+R)
  2. Wait 5 more minutes
  3. Check Deployments tab - should show "Ready"
  4. If still building, check build logs for errors

PROBLEM: Build fails on Vercel
SOLUTION:
  1. Click on failed deployment
  2. Go to "Logs" tab
  3. Look for error message
  4. Common issues:
     - Missing dependencies → npm install locally first
     - Wrong Node version → Check in Settings
     - Missing environment variables → Add in Settings

PROBLEM: App won't load after deployment
SOLUTION:
  1. Check if it's responding at all
  2. Open browser DevTools (F12)
  3. Check Console tab for errors
  4. Common issues:
     - API not connected → backend not deployed
     - Environment variables missing → add them
     - Port issues → Vercel uses port 3000 automatically

PROBLEM: Git push fails
SOLUTION:
  If you get "permission denied" or "authentication failed":
  1. Generate GitHub personal token:
     - Go to https://github.com/settings/tokens
     - Click "Generate new token"
     - Select: repo, workflow
     - Copy the token
  2. Try push again
  3. Paste token when asked for password (not your GitHub password)

PROBLEM: "fatal: remote already exists"
SOLUTION:
  git remote remove origin
  git remote add origin https://github.com/nagaraju-42/slotcut-app.git
  git push -u origin main
