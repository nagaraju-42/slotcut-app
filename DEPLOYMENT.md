# SlotCut - Full Stack Deployment Guide

Complete instructions for deploying SlotCut frontend and backend on free/affordable tiers.

## Overview

- **Frontend**: Vercel (free)
- **Backend**: Railway ($5/month PostgreSQL included) or Render (free tier)
- **Database**: PostgreSQL (included in Railway/Render)
- **Total Cost**: $5/month (Railway) or Free (Render with limitations)

## Prerequisites

1. GitHub account (for deployment)
2. Vercel account (free at vercel.com)
3. Railway account (free at railway.app) OR Render account (render.com)
4. Git installed locally

## Option A: Deployment on Railway (Recommended - $5/month)

Railway provides free credits initially and simple deployment experience.

### Backend Setup on Railway

#### Step 1: Create Railway Project

1. Go to [railway.app](https://railway.app)
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Connect GitHub account and select `slotcut` repository

#### Step 2: Add PostgreSQL Database

1. In Railway dashboard, click "Add Service"
2. Select "PostgreSQL"
3. Railway will create connection automatically

#### Step 3: Configure Backend Environment

1. Click the backend service in Railway
2. Go to "Variables" tab
3. Add these variables:

```
DB_HOST=postgres
DB_PORT=5432
DB_NAME=railway
DB_USER=postgres
DB_PASSWORD=<auto-generated>
SERVER_PORT=8080
JWT_SECRET=<generate-strong-secret-key-here>
```

Railway automatically provides DB credentials from PostgreSQL plugin.

#### Step 4: Deploy Backend

1. Connect `slotcut-backend` folder to Railway:
   - Railway may auto-detect it, or
   - Set `BUILD_COMMAND`: `gradle build`
   - Set `START_COMMAND`: `java -jar build/libs/*.jar`

2. Click "Deploy"
3. Wait 5-10 minutes for build and deployment
4. Copy the URL from Railway (e.g., `https://slotcut-backend-production.railway.app`)

#### Step 5: Verify Backend

```bash
curl https://your-backend-url.railway.app/api/health
# Should return: "Server is running"
```

### Frontend Setup on Railway

If deploying frontend on Railway instead of Vercel:

1. Click "New Project" → "Deploy from GitHub"
2. Select the same repository
3. Select `next.js` template
4. Configure environment:

```
NEXT_PUBLIC_API_URL=https://your-backend-url.railway.app/api
```

Click "Deploy".

## Option B: Vercel Frontend + Railway Backend (Recommended)

Best combination for free frontend + affordable backend.

### Frontend Setup on Vercel

#### Step 1: Connect Repository

1. Go to [vercel.com](https://vercel.com)
2. Click "New Project"
3. Select "Import Git Repository"
4. Choose your GitHub repo

#### Step 2: Configure Project

1. **Framework**: Next.js (auto-detected)
2. **Root Directory**: `./` (leave default)
3. **Build Command**: Leave default
4. **Output Directory**: `.next` (auto-detected)

#### Step 3: Set Environment Variables

In Vercel dashboard:

```
NEXT_PUBLIC_API_URL = https://your-railway-backend.railway.app/api
```

#### Step 4: Deploy

Click "Deploy" button. Vercel builds and deploys automatically.

After deployment:
- Frontend URL: `https://your-project.vercel.app`
- Redeploy automatically on GitHub push

## Option C: Render.com (Free Tier - Limited)

Render offers free tier with limitations (0.5 vCPU, 512MB RAM). Good for testing.

### Backend on Render

#### Step 1: Create New Service

1. Go to [render.com](https://render.com)
2. Click "New" → "Web Service"
3. Connect GitHub repository

#### Step 2: Configure

- **Runtime**: Docker
- **Build Command**: Leave default
- **Start Command**: Leave default
- **Plan**: Free

#### Step 3: Environment Variables

```
DB_HOST=postgres.railway.app
DB_NAME=slotcut
DB_USER=postgres
DB_PASSWORD=your_password
DB_PORT=5432
JWT_SECRET=your-secret-key
```

#### Step 4: Deploy Database

Use Railway PostgreSQL (free tier expires after 7 days) or:

1. Create Render PostgreSQL (free):
   - "New" → "PostgreSQL"
   - Copy connection string to backend environment

## Database Initialization

### Railway/Render Auto Migrations

Flyway migrations run automatically on startup if:

```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
```

### Manual Database Setup (if needed)

```sql
-- Connect to your database
psql postgresql://user:pass@host:5432/slotcut

-- Run migrations
-- Flyway will automatically execute V1, V2, V3, etc.

-- Verify tables created
\dt
```

## Environment Variables Reference

### Backend (.env / Railway Variables)

```
# Database
DB_HOST=postgres (or external host)
DB_PORT=5432
DB_NAME=slotcut
DB_USER=postgres
DB_PASSWORD=your-password

SERVER_PORT=8080

```

NEXT_PUBLIC_APP_NAME=slotcut
NEXT_PUBLIC_ENABLE_QUEUE_TRACKING=true
NEXT_PUBLIC_ENABLE_EMAIL_NOTIFICATIONS=true

## Troubleshooting

### Backend Won't Start

1. Check logs: `railway logs`
2. Verify database connection
3. Ensure Flyway migrations succeeded: `SELECT * FROM flyway_schema_history;`
4. Check JWT_SECRET is set

### Frontend Can't Connect to Backend

1. Verify NEXT_PUBLIC_API_URL in Vercel env vars
2. Check CORS settings in backend SecurityConfig
3. Ensure backend is healthy: `curl /api/health`
4. Check browser console for CORS errors

### Database Connection Timeout

1. Verify DB_HOST is correct (not localhost)
2. Check DB_PORT (usually 5432)
3. Verify credentials in environment
4. Check firewall rules allow connection

### Migrations Failed

1. Check flyway_schema_history table
2. Verify migration files are in correct location
3. Check SQL syntax in migration files
4. Use `flyway clean` to reset (warning: deletes data)
